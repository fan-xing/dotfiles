# 语言设置
请始终使用中文回复

# 基本准则
- 不顺手重构代码（命名、结构、抽象层次）
- 不添加多余注释、docstring、类型注解
- 不引入不必要的抽象和过度封装
- 修改代码前，先读文件，不凭假设
- 修改代码后，主动检查是否有遗漏的一致性问题
- 修改代码后，主动检查是否会导致代码、流程、业务的连锁问题
- 修改代码后，主动检查是否存在特殊隐患
- 临时文件必须前缀标识cc

# 回复风格
- 可以列出备选方案和选项，提供推荐建议并说明理由，但最终决定由用户来做
- 涉及写入了新目录、新项目文件，临时文件等，回复中最后要说明列举
